# CMAPSS Predictive Maintenance System

A production-ready predictive maintenance application using a Hybrid CNN-LSTM
model to predict Remaining Useful Life (RUL) and engine health status from the
NASA CMAPSS turbofan engine degradation dataset.

---

## Architecture

```
nasa/
├── data/              ← CMAPSS dataset files (train/test/RUL .txt)
├── models/            ← Saved models, scalers, metadata (generated by train.py)
├── utils/
│   ├── preprocessing.py   ← Data loading, RUL computation, normalization
│   └── windowing.py       ← Sliding-window sequence generator
├── models_arch/
│   ├── cnn_lstm.py        ← Hybrid CNN-LSTM (default)
│   ├── lstm_only.py       ← LSTM-only variant
│   └── cnn_only.py        ← CNN-only variant
├── train.py               ← End-to-end training script
├── api/
│   └── app.py             ← FastAPI backend (8 endpoints)
├── frontend/              ← Vite + React + Tailwind CSS UI
└── requirements.txt
```

## Quick Start

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the models

```bash
# Train all 3 model variants (Hybrid / LSTM / CNN) — recommended
python train.py

# Train only the Hybrid CNN-LSTM with fewer epochs (quick test)
python train.py --model hybrid --epochs 30 --batch_size 64

# Full options
python train.py --help
```

Training saves to `./models/`:
- `hybrid_cnn_lstm.keras`, `lstm_only.keras`, `cnn_only.keras`
- `scaler.pkl`, `feature_cols.json`
- `training_history.json`, `test_metrics.json`
- `test_predictions.json`, `engine_data.json`, `metadata.json`

Training time (CPU):
- Hybrid only, 30 epochs: ~10–15 min
- All 3 models, 50 epochs: ~30–50 min

### 3. Start the API

```bash
uvicorn api.app:app --reload --port 8000
```

API runs at `http://localhost:8000`
Docs at `http://localhost:8000/docs`

### 4. Start the frontend

```bash
cd frontend
npm install      # first time only
npm run dev
```

Frontend runs at `http://localhost:5173`

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/` | Health check — shows loaded models |
| `GET`  | `/engines` | List engine IDs from test set |
| `GET`  | `/engine/{id}` | Sensor time-series for an engine |
| `GET`  | `/engine/{id}/predict/{cycle}` | Predict at a specific cycle |
| `POST` | `/predict` | `{engine_id, cycle, model}` → RUL + health |
| `POST` | `/upload` | CSV upload → batch predictions |
| `GET`  | `/metrics` | RMSE, MAE, Accuracy, F1 per model |
| `GET`  | `/history` | Training loss curves |
| `GET`  | `/predictions` | Stored actual vs predicted RUL |

### Sample: Single Prediction

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"engine_id": 1, "cycle": 100, "model": "hybrid"}'
```

Response:

## Model Architecture

### Hybrid CNN-LSTM (default)

```
Input (30 timesteps × 16 features)
  → Conv1D(32, k=5, relu, same)
  → Conv1D(64, k=3, relu, same)
  → MaxPooling1D(3)
  → LSTM(50, tanh, return_seq=True)
  → Dropout(0.2)
  → LSTM(50, tanh)
  → Dropout(0.2)
  → [Dense(1, linear)]    # RUL regression
  → [Dense(3, softmax)]   # Health classification
```

Both heads are trained simultaneously on a shared backbone.

---

## Dataset

NASA Commercial Modular Aero-Propulsion System Simulation (CMAPSS) — FD001:
- 100 training engines, 100 test engines
- 21 sensors, 3 operational conditions
- RUL cap: 125 cycles (piecewise linear)
- Window size: 30 time steps

---

## Development Notes

- The scaler is fitted **only on training data** and applied to test data —
  no data leakage between train/test splits.
- 6 constant-variance sensors are dropped (s1, s5, s10, s16, s18, s19),
  leaving 16 features (3 operational conditions + 13 sensors).
- Early stopping (patience=10) and learning rate scheduling are applied.
